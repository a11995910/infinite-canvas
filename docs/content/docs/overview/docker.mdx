# Docker 部署

如果你希望在自己的机器或服务器上运行项目，可以直接使用 Docker Compose。主应用镜像包含 Vite 构建产物、Nginx 和 Go API：Nginx 对外提供前端路由回退，并把同源 `/api/*` 请求转发到容器内的 Go 服务。

## 使用发布镜像

```bash
git clone https://github.com/basketikun/infinite-canvas.git
cd infinite-canvas
cp .env.example .env
docker compose -f docker-compose.local.yml up -d --build
```

启动后访问：

```text
http://localhost:13000
```

## 本地构建镜像

默认 `docker-compose.yml` 将服务绑定到 `127.0.0.1:13000`，适合交由反向代理提供 HTTPS；本机直接访问时使用 `docker-compose.local.yml`。

```bash
docker compose -f docker-compose.local.yml up -d --build
```

## 文档站镜像

文档站位于 `docs/` 目录，按带服务端能力的 Next.js standalone 应用单独构建，不打进主应用镜像。

使用发布镜像：

```bash
cd docs
docker compose up -d
```

基于当前源码本地构建：

```bash
cd docs
docker compose -f docker-compose.local.yml up -d --build
```

## 服务端数据

Go API 默认使用 SQLite，在容器的 `/app/data` 下保存用户、画布同步数据、提示词和服务端存储索引。部署时应通过 Compose 为该目录挂载持久卷，并在 `.env` 中设置 `ADMIN_PASSWORD`、`JWT_SECRET` 和 `SUB2API_EMBED_PROXY_SECRET`。

浏览器中的模型渠道、AI API Key、素材和生成记录仍以本地存储为主；画布项目会在账号启用同步能力时与服务端画布快照同步。
