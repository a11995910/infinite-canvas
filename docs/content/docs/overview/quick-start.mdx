# 快速开始

如果你只是想先把项目跑起来，可以启动 Vite 前端；需要账号、画布服务端同步或 Sub2API iframe 嵌入时，还需要同时启动 Go API。

## Vercel 部署

在 Vercel 中导入仓库即可，根目录 `vercel.json` 会构建 `web/`。AI 请求由浏览器前台直连用户自己的 OpenAI 兼容地址；Vercel 静态部署不包含本仓库的登录、画布同步和 Sub2API API。

## 本地启动

```bash
git clone git@github.com:basketikun/infinite-canvas.git
cd infinite-canvas
cp .env.example .env
go run .
```

另开一个终端：

```bash
cd web
bun install
bun run dev
```

启动后访问：

```text
http://localhost:3000
```

## Docker 启动

如果你需要基于当前源码构建镜像：

```bash
cp .env.example .env
docker compose -f docker-compose.local.yml up -d --build
```

## 首次使用建议

- 先打开右上角配置弹窗，填入自己的 `Base URL`、`API Key` 和模型名。
- 如果需要提示词仓库内容，打开 `/prompts` 会由浏览器前端直连拉取并缓存到 IndexedDB。
- 如果需要跨设备同步画布、素材和生成记录，可在配置弹窗中填写 WebDAV。

## 说明

- 画布项目主要保存在浏览器本地；登录账号且服务端启用同步时，画布项目会同步到服务端。我的素材仍主要保存在浏览器本地，WebDAV 同步需要用户自行配置。
- AI API Key 保存在浏览器本地，并由前端直接请求 OpenAI 兼容接口。
