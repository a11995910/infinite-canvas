# 待测试

- Sub2API 嵌入生图代理允许请求运行到 900 秒；生产环境 Canvas 域名的 Nginx `proxy_read_timeout` 和 `proxy_send_timeout` 也必须至少为 900 秒，避免上游实际生图成功但站点在默认 60 秒先返回 `504`。前端收到 HTML 格式的 `504` 时会提示站点网关超时及不要立即重复提交，不再误报为 Base URL 错误。
- 生图工作台：OpenAI 兼容渠道会把站点根地址、`/v1` 地址、`llms.txt` / `llms-full.txt` 文档地址以及误填的 `/v1/models`、`/v1/images/generations`、`/v1/images/edits` 完整接口地址统一归一化为正确 Base URL；需分别使用 `https://api.skylee9.cloudns.ch`、`https://api.skylee9.cloudns.ch/v1` 和 `https://api.skylee9.cloudns.ch/llms.txt` 拉取模型，并用 `gpt-image-2` 验证文生图和图生图。
- 生图工作台：多张参考图允许部分上传成功并显示失败数量，上传期间禁止提交生成；新建会话会清理未被任务或历史记录引用的临时参考图。图片已经生成但本地/服务端持久化或生成记录写入失败时会保留当前结果卡片，便于用户立即下载；保存到素材时使用对应生成任务的提示词，不受当前输入框内容影响。
- 生图历史同步：启用账号数据同步后，登录会按记录 ID 合并远端和浏览器本地历史及分类，再把合并结果同步到账号，避免覆盖尚未上传的本地生成记录。
- 本地模型渠道支持选择接口格式：默认继续使用 OpenAI 兼容格式，也可为单个本地渠道切换为 Gemini；配置弹窗会按渠道格式拉取模型列表，图片生成、图生图和文本问答会按当前模型所在渠道分别走 OpenAI 兼容接口或 Gemini `generateContent` / `streamGenerateContent`。浏览器直连本地渠道遇到网络或 CORS 类失败时，会自动回退到同源本地代理访问模型列表、图片、编辑和文本接口；本地代理允许长图像任务运行到 900 秒，并会把上游连接失败返回为明确的代理错误；单渠道拉取模型时会默认选中本次接口真实返回的模型，历史模型仅作为可手动保留的已有候选；确认渠道模型后，如果默认生图、视频或文本模型已经不在该渠道真实模型列表中，会自动回退到该渠道第一个可用模型；本地渠道可打开诊断弹窗，只读取模型列表并检查默认生图、视频、文本模型是否在接口真实返回列表中，便于发现 `image2` 与 `gpt-image-2` 这类模型名不一致问题；图片响应头和响应体读取都会按请求超时时间兜底，且超时错误不会再被瞬时重试放大；生图工作台的生成中卡片会在模型已调用但上游排队、响应较慢或接近超时时给出轻量提示；上游返回 HTML 错误页、无效 Key、账号不支持模型、Key 分组未开启图片生成或图片接口不认可当前模型名时，失败卡片会显示更明确的中文原因并截断过长详情，远程后台渠道和 Sub2API 嵌入仍保持原有路径。
- 画布图片节点悬浮工具栏新增“迭代”入口：可打开图片迭代面板，按“生成变体、换风格、换背景、重构图、局部重绘”创建右侧新图片节点并自动连接原图；迭代结果会保留本次提示词、模型、尺寸、质量、参考图和生成耗时，生成失败只标记新节点，不覆盖原图。
- 画布图片节点迭代任务改为按节点独立记录运行状态：一张图片正在迭代时，其他图片仍可打开迭代面板并发起新的迭代任务；不同任务完成或失败时只清理自己的运行状态，不会互相影响。
- 图片迭代面板支持为当前图片标注参考角色，包括通用、主体、风格、构图、色彩、背景和锁定；该角色会写入图片节点元数据，并在后续配置节点或图片节点图生图时自动拼入参考图角色提示，减少主体漂移或风格误用。
- 图片迭代面板的“局部重绘”支持在原图上拖拽选择局部区域，并向 `/images/edits` 请求传递 `mask`；支持 mask 的上游模型会优先只修改透明选区，不支持 mask 或 Agnes 图生图类模型时仍会通过提示词强调局部修改，但最终约束能力取决于上游模型。
- 首页和提示词库会展示远程提示词源同步得到的封面与预览图；远程图片会通过 `/api/proxy-image` 代理加载，避免前端直连 GitHub raw 图片失败时整卡退回 Logo 占位。提示词详情会从预览 Markdown 中提取结果图并以图片网格展示，剩余文本仍保留在预览文本区。
- mageia `awesome-gpt-image-2-API-and-Prompts` 来源同步提示词时，系统会兼容上游 `x.com` / `twitter.com` 推文链接、结构化 `prompt_text` / `media_urls` 记录和 Markdown 案例记录；有 `image_dir` 时优先使用 GitHub raw 输出图，缺少 `image_dir` 时回退到媒体链接或 Markdown 图片，并将 `../images`、`../../images` 等相对路径归一为 GitHub raw 图片地址，避免源项目结构变化后同步结果不更新或封面链接无效。
- 画布图片节点悬浮工具栏新增“拆物件”入口：点击后会先用文本/视觉模型识别图中最多 8 个适合独立拆出的主体物件，再按识别到的 bbox 从原图裁剪出独立图片节点并连接到原图；识别阶段只接受结构化 JSON 清单，缺少 bbox 的子项会标记失败，不会调用生图模型重绘，也不会覆盖原图。当前版本是原图区域裁剪，不是透明背景抠图。
- Sub2API 自定义菜单 iframe 嵌入模式下，页面会读取 `ui_mode=embedded`、`token` 和 `src_host`，通过 Sub2API `/api/v1/auth/me` 校验当前用户并创建或复用画布本地会话，避免 iframe 内二次登录；登录成功后再通过服务端代理获取当前用户 API Key，并自动写入本地直连渠道。嵌入模式下的画布列表、项目详情、顶部导航、移动导航和登录页兜底都会保留 Sub2API 嵌入参数；如果换票失败，只显示 Sub2API 登录状态错误，不再跳转或展示画布自己的账号密码登录页。AI 请求会经 `/api/sub2api/proxy/*` 由画布服务端转发到 Sub2API，避免浏览器跨域直连。画布服务端生成代理地址时会优先使用反向代理传入的公开协议和域名，适配 HTTPS iframe 场景。
- Sub2API 嵌入模式下的配置弹窗改为简化视图，展示“生图 Key”和“文本 Key”两个选择器，以及图片模型、文本模型和视频模型预留选择；图片和视频请求使用生图 Key，文本问答、画布文本节点、工作流 Agent 和系列提示词草稿使用文本 Key。切换 Key 后会分别通过 Sub2API 代理刷新对应渠道模型列表，失败时保留默认模型兜底。
- 文本问答、画布文本节点、画布配置节点文本模式、工作流 AI 创建和多图工作流提示词规划都会按文本用途解析模型与渠道；图片节点、生图工作台和工作流出图阶段按图片用途解析模型与渠道；视频入口按视频用途解析模型与渠道。旧节点或旧工作流保存的模型不属于当前用途渠道时，会回退到对应用途渠道的可用模型。
- Docker 源码部署镜像的前端构建阶段改用 Node 22 执行 `npm ci --legacy-peer-deps` 和 `npm run build`，运行阶段仍由 Next.js 监听 `13000` 并代理内部 Go API，避免服务器上的 Bun 构建进程异常导致镜像无法上线。
- 生图配置已禁用 Responses 图像模式；配置弹窗、画布图片节点、生成配置节点、生图工作台和工作流编辑器不再展示图片协议选择，旧配置和旧历史记录会按 Images 模式处理，真正的文本对话模式不受影响。
- 生图参数新增输出格式、压缩比例和审核级别；画布节点、配置节点和助手生图会随请求传递 `output_format` 等参数。
- 生图工作台支持连续提交多个并发生图任务，结果卡片展示任务时间、提示词、模型、尺寸、质量、格式和耗时，失败任务展示上游报错并支持单任务重试。
- 生图工作台将生成记录并入生成结果区展示，结果卡片支持提示词展开和复制；生成中卡片实时显示耗时，参考图区域支持展开/收起且收起时保留已选参考预览。
- 生图记录保存生成接口返回的内联图片作为本地存储回退，避免离开生图工作台后历史结果图片无法显示。
- 生图工作台新增侧边/底部工作台切换并持久化选择；提示词、参考图、参数均支持展开/收起摘要展示，历史结果卡片改为紧凑网格。
- 无限画布图片节点支持双击查看大图；生成中的图片节点会显示实时耗时，生成完成或失败后记录本次耗时。
- 前端云端渠道配置会展示后台渠道摘要；当公开可用模型未手动勾选时，会回退使用已启用私有渠道的模型列表，避免云端模式无模型可选。
